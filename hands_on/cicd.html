🔄 CI/CD Pipeline in DevOps

CI/CD (Continuous Integration and Continuous Deployment) is a core DevOps practice that automates the process of building, testing, and deploying applications. It helps teams deliver code changes faster and more reliably by reducing manual intervention.

With CI/CD pipelines, developers can continuously integrate code, run automated tests, and deploy applications to production environments efficiently, ensuring faster releases, improved quality, and consistent delivery.

🔹 GitLab CI/CD Pipeline

I implemented this GitLab CI/CD task to automate deployment of my resume applications using Docker containers. First, I created a GitLab repository named “two_index_files” and added my source code along with a Dockerfile for containerization. Then, I set up a self-hosted GitLab Runner named “two-index-runner” on my Ubuntu VM to execute pipeline jobs. After that, I wrote a .gitlab-ci.yml file to define the CI/CD pipeline for building and deploying the containers. Using this pipeline, I deployed two resume files (mapped as index.html and index1.html) inside Docker containers. Finally, I exposed them on localhost ports 8001 and 8002, successfully achieving automated deployment through GitLab CI/CD.

Tools Used

  • GitLab CI/CD

Prerequisites

  • GitLab account
  • Project setup
  • Access to GitLab CI/CD features

🔹 GitHub Actions CI/CD Pipeline

I implemented this GitHub Actions setup as part of my CI workflow integrated with an ArgoCD pipeline. First, I created a GitHub repository and pushed my application source code to it. Then, I configured a GitHub Actions workflow by creating a django.yml file inside the .github/workflows/ directory. After that, I set up a self-hosted runner named “argocd” on my Ubuntu VM to execute the CI jobs. Whenever I pushed changes to the repository, the workflow automatically built a Docker image and pushed it to DockerHub. I securely stored DockerHub credentials (DOCKERHUB_USERNAME and DOCKERHUB_TOKEN) in GitHub Secrets, ensuring safe authentication during the image push process. This setup enabled automated CI and seamless integration with my deployment pipeline.

Tools Used

  • GitHub Actions

Prerequisites

  • GitHub account
  • Repository setup
  • Access to GitHub Actions features

🔹 CircleCI CI/CD Pipeline

I implemented this CircleCI task to automate the deployment of my resume application using a CI pipeline. First, I created an account on CircleCI and connected it with my GitHub repository containing the source code. Then, I set up a self-hosted runner on CircleCI and registered it with my Ubuntu VM to execute the pipeline jobs. After that, I triggered the pipeline by committing changes to my GitHub repository. The pipeline built and deployed the application inside a container, which I successfully exposed on localhost port 8086. This setup allowed me to achieve automated deployment using CircleCI.

🔹 ArgoCD Pipeline

I implemented this deployment by integrating Kubernetes with GitOps using ArgoCD. First, I deployed my Django-based user authentication application on a K3s Kubernetes cluster. Then, I configured ArgoCD to track my Git repository and manage the application deployment automatically. Once connected, ArgoCD successfully synchronized the application state with the repository, and the application status showed as healthy and fully synced. The Kubernetes resources were created and running correctly within the cluster. Finally, I exposed the application using a NodePort service on port 30080, allowing me to access it locally and verify that the deployment was working as expected.
