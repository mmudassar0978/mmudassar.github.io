🚀 Role of Ansible in DevOps

Ansible plays a key role in DevOps by enabling automation of configuration management, application deployment, and infrastructure provisioning. It uses YAML-based playbooks to define tasks, making it easy to manage complex environments. Its agentless architecture ensures fast and consistent deployment across systems.

🛠️ Project Implementation (Ansible Automation)

I implemented this project using Ansible to automate the deployment of multiple containerized services in a structured way. Instead of relying on default system configurations, I created a custom inventory file and ansible.cfg at the project level, which allowed me to manage hosts and configurations independently from the /etc/ansible directory. This made the project more portable and easier to version-control. I defined Ansible playbooks and roles to install Docker and manage all required services. Each service (PostgreSQL, Redis, Zookeeper, and NATS) was deployed using its respective Docker image with proper port mappings. I also included RedisInsight and pgAdmin to provide graphical interfaces for Redis and PostgreSQL monitoring. All services were orchestrated through Ansible tasks, ensuring consistency and repeatability in deployment. I verified that each container was running correctly and accessible through its assigned port. The project structure was kept modular so new services can be easily added in the future. Overall, this implementation helped me gain practical experience in automation, containerization, and infrastructure management using Ansible in a real DevOps workflow.

🔧 Services Overview

PostgreSQL is used as the main relational database for structured data storage and is managed via pgAdmin UI.

Redis is used for caching and fast data retrieval, monitored using RedisInsight UI.

Zookeeper is used for coordination and configuration management in distributed systems.

NATS is a messaging system used for service communication and runs on port 8222.
